# S2 UI Style Guide v0.1

這份文件是 `excel_game_s2` 的 UI 基準。它不是最終美術規格，而是之後所有 UI mockup、Godot 介面、按鈕、彈窗、解謎畫面都要跟著走的共同語言。

---

## 1. 核心方向

S2 的 UI 關鍵詞：

- 貴族偵探
- 魔法資料學
- 半寫實
- 安靜懸疑
- Excel 解謎器
- 成熟遊戲介面

UI 看起來應該像主角自己設計的工具：冷靜、精準、漂亮，但不炫耀魔法。

### 避免方向

- 可愛 anime
- 現代辦公軟件
- cyberpunk hologram
- 華麗魔法少女
- 普通 RPG 道具欄

---

## 2. 主色規則

### 主要顏色

- 深炭黑：主要 UI 底色、對白框、面板背景
- 深綠：數據計算儀、選取狀態、重要互動提示
- 銀色：邊框、分隔線、貴族感細節
- 象牙白：文字、紙張、案件文件感
- 暗酒紅：少量用於警告、重要線索、案件危險感

### 顏色限制

- 不要讓整個 UI 變成單一深藍。
- 不要讓整個 UI 變成單一紫色。
- 不要讓整個 UI 變成全黑綠。
- 紫色只用來呼應主角眼睛，不作為 UI 主色。

---

## 3. 材質感

UI 材質應該像：

- 磨砂黑玻璃
- 深色皮革
- 銀製細框
- 象牙色案件紙
- 淡綠魔法數據光

不要像：

- 透明科幻螢幕
- 現代手機 app
- 普通 Excel 白底介面
- 過度華麗的魔法陣

魔法感要克制，像「資料正在被計算」，不是「施放大招」。

---

## 4. 文字規則

正式遊戲所有文字都由 Godot UI 顯示，不寫死在背景圖或 mockup 圖像裡。

主要 UI 文字使用繁體中文。公式名稱可以保留英文，例如：

- COUNTIF
- SUMIFS
- XLOOKUP
- LEFT
- MID
- DATE

### 常用按鈕文字

- 開始調查
- 繼續調查
- 新的案件簿
- 保存調查
- 讀取檔案
- 設定
- 返回調查
- 數據計算儀
- 案件目標
- 公式提示

### 語氣規則

文字語氣要像偵探系統，不要太像遊戲教學。

適合：

> 目前請使用 COUNTIF 檢查證言狀態。

不適合：

> 你還不能使用這個功能喔！請先完成教學！

更適合的限制提示：

> 這項操作與目前調查無關。先確認案件所需的資料吧。

---

## 5. 按鈕風格

按鈕形狀：

- 低調矩形
- 小圓角
- 銀色細邊
- 深色底
- hover / selected 狀態使用淡綠光

重要按鈕可以稍大，但不要像手機遊戲的大膠囊按鈕。

### 主選單按鈕層級

- 開始調查：最大、最明顯
- 繼續調查：次要
- 讀取檔案、設定、離開：更安靜

### 遊戲中小按鈕

遊戲中小按鈕要 compact，例如：

- 保存
- 讀取
- 設定
- 紀錄
- 自動

---

## 6. Story Dialogue UI

用途：劇情、角色對話、案件開場、重要推理。

### 畫面結構

- 背景全螢幕。
- 角色立繪放中右或中左。
- 下方 25-30% 是對白框。
- 姓名框附在對白框上方。
- 右上角放保存、讀取、設定、紀錄、自動。
- 案件目標可小型收合，不必一直大面積顯示。

### 對白框風格

- 深炭黑半透明
- 銀色細框
- 淡綠細節線
- 文字象牙白
- 重要詞可用淡綠或暗酒紅，但不要太多

---

## 7. Excel Solver / 數據計算儀

用途：核心解謎器。

### 畫面結構

- 上方：案件名、章節、保存、讀取、設定、返回
- 左側：案件資料分類
- 中央：表格、公式列、選取格
- 右側：案件目標、公式提示
- 下方：系統或角色提示

### 固定原則

- 案件目標必須常駐。
- 本關可用公式必須顯示。
- 未支援或非本關操作要用角色語氣引導。
- 表格要像 Excel，但不能完全像 office 軟件。
- 選取格、公式列、結果高亮都用淡綠。

### 公式提示

支援公式可以像工具清單一樣放在右側，例如：

- COUNTIF
- SUMIFS
- XLOOKUP
- LEFT
- MID
- DATE

### 錯誤提示語氣

玩家輸入錯誤時，不說「功能未完成」。

適合：

> 目前的資料不足以使用這個公式。

> 這份案件暫時不需要該計算方式。

> 先確認證言狀態欄，再進行統計。

---

## 8. Map Walker / 偵探移動畫面

用途：地圖探索、搜證、移動、觸發對話。

### 畫面結構

- 全螢幕場景。
- 小比例角色 avatar。
- 可調查點用淡綠 glint 或 magnifier marker。
- 左上：案件目標。
- 左下：地點名。
- 右上：地圖、數據計算儀、保存、設定。
- 右下：小地圖。
- 靠近線索時顯示「調查：xxx」。

地圖感覺要像真實地點，不是棋盤。不要一直顯示格線。玩家可以走路、靠近、調查、取得資料。

### 基本循環

探索地圖 -> 調查線索 -> 收集資料 -> 打開數據計算儀 -> 解出矛盾 -> 推進案件

---

## 9. Save / Load UI

用途：保存與讀取共用畫面。

### 畫面結構

- 左側：保存調查 / 讀取檔案
- 中央：案件檔案 slot
- 每個 slot 顯示章節、案件名、地點、狀態、時間
- 空 slot 顯示「空白檔案」
- 底部：返回 / 確認

新的案件簿只應該出現在主選單或讀取畫面，不一定要出現在遊戲中保存畫面。

視覺可以像案件檔案櫃、調查紀錄卡，不要像普通 save menu。

---

## 10. Settings UI

設定畫面分成三類。

### 音訊

- BGM 音量
- 音效音量

### 顯示與文字

- 文字速度
- 自動播放速度
- 對白框透明度
- 全螢幕
- 解析度
- 動畫特效

### 解謎輔助

- 公式提示
- 錯誤提示詳細度
- 高亮可互動格子

### 不需要的設定

- 是否略過已讀文字
- 是否顯示案件目標

案件目標是核心，不應讓玩家關掉。

---

## 11. 一致性規則

所有畫面都要有這幾個共同元素：

- 深色底
- 銀色細線
- 淡綠互動提示
- 象牙白文字
- 案件文件 / 偵探工具感
- 少量魔法資料光

### 各模式差異

- 主選單：最有氣氛，像偵探所開門前。
- Story：最重角色和情緒。
- Map Walker：最重空間和線索。
- Excel Solver：最重資料和計算。
- Save / Load：最重案件檔案感。
- Settings：最安靜、最功能性。
