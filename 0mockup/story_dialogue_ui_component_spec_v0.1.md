# S2 Story Dialogue UI Component Spec v0.1

這份文件定義 `excel_game_s2` 第一個正式 UI 零件：Story Dialogue UI Component。

這個 component 是故事模式的基礎，用於劇情對話、旁白、角色登場、案件開場、推理前後的情緒段落。它不是完整遊戲系統，而是之後 Chapter 1 可以直接接上的第一個穩定零件。

---

## 1. Component 目的

Story Dialogue UI Component 負責：

- 顯示背景圖。
- 顯示角色立繪。
- 顯示角色名字。
- 顯示對白文字。
- 點擊或按鍵進入下一句。
- 顯示右上功能按鈕。
- 顯示小型案件目標面板。
- 支援旁白、角色對白、系統提示。

它不負責：

- 案件邏輯判定。
- Excel Solver 計算。
- Map Walker 移動。
- Save / Load 實際資料寫入。
- 角色動畫完整系統。

---

## 2. 畫面結構

### 基本 layout

- 背景層：全螢幕。
- 角色層：背景之上，UI 之下。
- 對白框：畫面下方 25-30%。
- 姓名框：附在對白框左上方。
- 右上功能按鈕：保存、讀取、設定、紀錄、自動。
- 案件目標小面板：右側或左上角，可收合。

### 視覺比例

- 背景永遠覆蓋整個畫面。
- 角色立繪不可遮住姓名框和主要對白文字。
- 對白框不可超過畫面高度 1/3。
- 右上按鈕保持 compact，不搶走劇情焦點。
- 案件目標面板只顯示目前最重要的 1-3 個目標。

## 3. 子零件拆分

### 3.1 BackgroundLayer

用途：顯示當前場景背景。

需要支援：

- 更換背景圖。
- 淡入淡出轉場。
- 之後可支援短動畫或影片背景。

v0.1 只需要：

- 顯示一張靜態背景。
- 可切換到下一張背景。

---

### 3.2 CharacterSpriteLayer

用途：顯示角色立繪。

需要支援：

- 顯示一位或多位角色。
- 設定角色位置：left / center / right。
- 設定角色表情差分。
- 設定角色顯示、隱藏。

v0.1 只需要：

- 同時顯示最多 2 位角色。
- 支援 left / right 位置。
- 支援基本表情切換。

---

### 3.3 DialogueBox

用途：顯示主要對白文字。

視覺規則：

- 深炭黑半透明底。
- 銀色細邊。
- 淡綠細節線。
- 象牙白文字。

需要支援：

- 角色對白。
- 旁白。
- 系統提示。
- 點擊下一句。

v0.1 只需要：

- 一次顯示一句文字。
- 點擊後顯示下一句。
- 暫時不需要打字機效果。

---

### 3.4 NamePlate

用途：顯示說話者名字。

規則：

- 有角色說話時顯示角色名。
- 旁白時可以隱藏，或顯示「旁白」。
- 系統提示時可以顯示「系統」。

v0.1 建議：

- 角色對白：顯示角色名。
- 旁白：隱藏 NamePlate。
- 系統提示：顯示「調查紀錄」。

---

### 3.5 TopRightMenu

用途：提供故事模式中的基本功能入口。

按鈕：

- 保存
- 讀取
- 設定
- 紀錄
- 自動

v0.1 功能：

- 按鈕先有 UI 位置。
- 保存、讀取、設定可以先打開 placeholder 畫面或暫時無反應。
- 紀錄可以先記錄當前 session 的已讀對白。
- 自動可以先不實作，只保留位置。

---

### 3.6 CaseObjectiveMiniPanel

用途：在故事模式中顯示當前案件目標。

規則：

- 案件目標不可被玩家永久關閉。
- 故事模式可以收合。
- Excel Solver 裡必須常駐。

v0.1 顯示內容：

- 案件目標標題。
- 目前 1-3 個目標。
- 完成目標可打勾或變暗。

例子：

- 尋找第一位委託人
- 確認委託人的證言
- 前往白塔街

---

### 3.7 DialogueController

用途：控制對白資料播放。

需要處理：

- 目前對白 index。
- 下一句。
- 換背景。
- 換角色。
- 換表情。
- 更新案件目標。
- 對白結束後觸發下一個 scene 或 game mode。

v0.1 只需要：

- 順序播放對白。
- 點擊或按鍵下一句。
- 到結尾時發出 finished event。

---

## 4. 對白資料格式

之後 Godot 可以用 JSON、CSV、Resource 或自訂資料格式。v0.1 先用容易讀的結構規劃。

### 單句資料欄位

每一句對白建議包含：

- id：句子編號。
- type：dialogue / narration / system。
- speaker：說話者 id。
- speaker_name：畫面顯示名字。
- text：對白內容。
- background：需要切換背景時填入。
- characters：本句要顯示的角色和位置。
- expression：角色表情。
- objective_update：是否更新案件目標。
- next：下一句 id。

### 範例

```json
{
  "id": "ch1_001",
  "type": "dialogue",
  "speaker": "host",
  "speaker_name": "莉希雅",
  "text": "第一份委託，終於走到我的門前了。",
  "background": "detective_office_dusk",
  "characters": [
    {
      "id": "host",
      "position": "right",
      "expression": "calm"
    }
  ],
  "objective_update": null,
  "next": "ch1_002"
}
```

## 5. 操作方式

### 滑鼠 / 觸控

- 點擊對白框：下一句。
- 點擊畫面空白處：下一句。
- 點擊右上按鈕：打開對應功能。
- 點擊案件目標面板：收合或展開。

### 鍵盤

- Space：下一句。
- Enter：下一句。
- Esc：打開設定或暫停選單。

### 手把

v0.1 暫不需要，但 layout 不應阻礙未來加入。

---

## 6. 對白類型

### 角色對白

有 speaker_name，顯示 NamePlate。

例子：

> 莉希雅：第一份委託，終於走到我的門前了。

### 旁白

沒有 speaker_name，隱藏 NamePlate。

例子：

> 雨聲落在白塔街的石板路上，偵探所的門鈴第一次響起。

### 系統提示

顯示「調查紀錄」或其他系統名。

例子：

> 調查紀錄：新的案件目標已更新。

---

## 7. 案件目標規則

故事模式中，案件目標是輔助資訊，不應干擾角色演出。

規則：

- 預設顯示小型面板。
- 玩家可以暫時收合。
- 更新目標時短暫亮起淡綠提示。
- 完成目標時顯示勾選或變暗。
- 進入 Excel Solver 時必須展開並常駐。

---

## 8. 轉場規則

v0.1 需要支援的轉場：

- 背景淡入淡出。
- 角色淡入淡出。
- 對白框出現 / 隱藏。

暫時不需要：

- 複雜鏡頭推拉。
- Live2D。
- 影片插入。
- 大量角色動畫。

---

## 9. 和其他系統的連接

### 連到 Main Menu

玩家按「開始調查」後，進入 Story Dialogue UI 的第一句。

### 連到 Save / Load

故事模式需要提供保存與讀取入口，但 v0.1 可先只保留 UI 位置。

### 連到 Map Walker

一段故事結束後，可以進入探索模式。

例子：

故事對話結束 -> finished event -> Map Walker 載入「偵探所門前」。

### 連到 Excel Solver

當劇情需要使用數據計算儀時，可以跳轉到 Excel Solver。

例子：

故事對話結束 -> finished event -> Excel Solver 載入「證言表」。

---

## 10. v0.1 範圍

v0.1 要做：

- 靜態背景顯示。
- 最多 2 位角色立繪。
- 姓名框。
- 對白框。
- 點擊 / Space / Enter 下一句。
- 右上功能按鈕位置。
- 案件目標小面板。
- 對白結束 event。

v0.1 不做：

- 完整 Save / Load。
- 完整 Settings。
- 完整 Auto Mode。
- 完整 Text Log。
- 語音。
- 影片。
- Live2D。
- 複雜角色動畫。
- 分支選項。
- Excel Solver 內部邏輯。

## 12. 第一個 Demo 使用方式

Chapter 1 故事完成後，第一個 demo 可以用這個 component 做：

1. 主選單按「開始調查」。
2. 進入偵探所背景。
3. Host 獨白。
4. 第一位 main character 登場。
5. 對話引出第一個案件。
6. 更新案件目標。
7. 對話結束後進入 Map Walker 或下一段故事。

這樣 demo 不需要先完成所有系統，也可以讓玩家感受到 S2 的核心氣質。
